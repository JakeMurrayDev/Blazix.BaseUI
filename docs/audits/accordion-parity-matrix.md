# Accordion Parity Matrix

Date: 2026-06-26

| React source surface | Required behavior | Blazor equivalent | Verification |
| --- | --- | --- | --- |
| `Accordion.Root` | Renders default `div`; supports `value`, `defaultValue`, `disabled`, `hiddenUntilFound`, `keepMounted`, `multiple`, deprecated `loopFocus`, `orientation`, `onValueChange`, state callbacks, and render override. | `AccordionRoot.razor` with controlled/uncontrolled `TValue[]`, `RenderElement<AccordionRootState<TValue>>`, `ClassValue`, `StyleValue`, `Render`, inherited mount/search options, and cancelable value changes. | Accordion bUnit tests, Playwright tests, source docs comparison. |
| `useControlled` | Controlled value does not mutate internally without a parameter update. | `CurrentValue => IsControlled ? Value! : currentValue`; uncontrolled state only mutates when not controlled. | `ControlledValueDoesNotMutateWithoutParameterUpdate`. |
| `useStableCallback` value change handler | Event details can cancel before state commit. | `AccordionValueChangeEventArgs<TValue>.Cancel()` stops uncontrolled state mutation and `ValueChanged`. | `ItemOnOpenChangeCancellationPreventsRootValueChange`. |
| Root state attributes mapping | Root `value` maps to no open/closed data attribute. | Root emits orientation/disabled/root marker only; no root `data-open`/`data-closed`. | Static source comparison and in-app browser evidence. |
| `AccordionRootContext` | Shares disabled, value, state, hidden-until-found, keep-mounted, and value-change handler. | `AccordionRootContext<TValue>` updates native Blazor context state in `OnParametersSet`. | bUnit root/item/panel integration tests. |
| `Accordion.Item` | Resolves fallback value, derives open state from root value, combines root/item disabled state, and exposes item state attributes. | `AccordionItem.razor` resolves value, registers index, derives `IsOpen`, emits item `data-open`/`data-closed`/`data-disabled`, and exposes state. | Item attribute tests and Playwright state tests. |
| `useBaseUiId` fallback | Item, trigger, and panel IDs remain stable and recover when manual IDs are removed. | `AccordionItem` creates GUID-based IDs and trigger/panel setters restore generated IDs on null. | Manual ID tests in `AccordionRootTests`. |
| `Accordion.Item onOpenChange` | Cancellation prevents root value propagation. | `HandleTrigger` and `HandleBeforeMatch` invoke item `OnOpenChange` first and stop when canceled. | `ItemOnOpenChangeCancellationPreventsRootValueChange` and beforematch cancellation tests. |
| `useCollapsibleRoot` | Accordion item delegates open/close and mounted state to Collapsible semantics. | `CollapsibleRootContext` bridge carries open, disabled, transition status, panel ID, trigger handler, beforematch handler, and open-change reason. | bUnit JSInterop tests and Playwright hidden/mount tests. |
| `Accordion.Header` | Renders default `h3` and forwards attributes. | `AccordionHeader.razor` renders `h3` through `RenderElement`. | Header bUnit and Playwright tests. |
| `Accordion.Trigger` | Renders default button, uses `useButton`, remains focusable when disabled, and suppresses disabled activation. | `AccordionTrigger.razor` resolves native button attributes and disabled ARIA/data attributes; JS handles keyboard activation. | Trigger disabled tests and Playwright disabled tests. |
| `useButton` Space behavior | Space keydown prevents scroll; Space keyup activates. | `blazix-baseui-accordion-trigger.js` prevents default on Space `keydown` and clicks on Space `keyup`. | `SpaceKeyTogglesOnKeyUp` Playwright test. |
| Non-native trigger behavior | Enter activates non-native trigger on keydown; Space activates on keyup. | Trigger JS retains non-native Enter click and Space keyup click. | Existing keyboard Playwright coverage and source comparison. |
| `aria-controls` | Trigger references panel only while open. | Trigger emits `aria-controls` when `IsOpen` and panel ID exists. | Playwright and bUnit attribute tests. |
| Trigger manual `id` | Panel `aria-labelledby` tracks manual trigger ID and restores generated ID after removal. | Trigger context setter updates `AccordionItem.TriggerId`; null restores generated ID. | `UpdatesPanelAriaLabelledByWhenManualTriggerIdChanges` and related tests. |
| `Accordion.Panel` | Renders default `div`, role `region`, `aria-labelledby`, hidden state, and open/closed data attributes. | `AccordionPanel.razor` renders panel with inherited item/root state and shared Collapsible JS lifecycle. | Panel bUnit tests and Playwright DOM tests. |
| Panel presence condition | Render when `keepMounted || hiddenUntilFound || mounted || open`. | `AccordionPanel.IsPresent` implements the equivalent with root inheritance and panel override. | `InheritsKeepMountedFromRoot` and `InheritsHiddenUntilFoundFromRootAndAllowsPanelOverride`. |
| `hiddenUntilFound` | Closed searchable panel remains mounted with `hidden="until-found"` and opens from `beforematch`. | Panel registers `beforematch`; item handles reason `None`; panel passes instant-open flag to JS. | `BeforeMatchOpenPassesInstantOpenFlagToJavaScript` and Playwright hidden-until-found tests. |
| Collapsible panel animation utilities | DOM measurement and transition status remain in JS. | Shared `blazix-baseui-collapsible.js` handles dimensions, starting/ending style, hidden cleanup, and no-motion close completion. | Accordion Playwright animation tests. |
| `BaseUIChangeEventDetails` reasons | Trigger changes use trigger reason; beforematch uses none reason. | `AccordionValueChangeReason.TriggerPress` and `None`; `CollapsibleOpenChangeReason` bridge for panel JS. | bUnit event detail tests. |
| `useRenderElement` | Render override receives current component state and attributes. | All Accordion parts use `RenderElement<TState>` with state, attributes, class/style callbacks, and render fragments. | bUnit render/state callback tests. |
| Source docs | Docs expose Root, Item, Header, Trigger, Panel and multiple-panel example. | Blazor test page exposes equivalent parts and tested states. | Source docs PNPM validation and in-app browser comparison JSON. |

No React hook, utility, or upstream fix evaluated in this audit remains unaccounted for.
